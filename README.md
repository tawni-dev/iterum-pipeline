# iterum-pipeline

Public Reference Authority for [Iterum](https://github.com/tawni-dev/iterum) projects.

## What this is

This repository contains `canon.json` — the approved registry of GitHub Action commit SHAs, security tool versions, container base image digests, and backend dependency versions used by all projects generated with Iterum.

Every Iterum project fetches this file at bootstrap and at CI runtime before any other step. If the fetch fails, everything fails. No project repo contains a copy of canon.json. Under the documented non-admin trust model, agents may propose changes through PRs but cannot merge them.

**Canon URL:**

```
https://raw.githubusercontent.com/tawni-dev/iterum-pipeline/main/canon.json
```

This repo is public, so Iterum projects fetch from it with no authentication required.

## Reference Authority

This repository is intended for learning, evaluation, demonstrations, and prototypes. Projects may consume its public canon URL, but long-lived and production projects should create and maintain their own pipeline authority and `canon.json`. The authority is intentionally replaceable by changing the configured canon URL.

Projects that continue trusting this Reference Authority are downstream consumers of its integrity. Compromise of the maintainer account or this repository could affect consumers that have not migrated to their own authority. This is a documented trust-model risk, not a secret or vulnerability in the public contents of `canon.json`.

## Setup

See `FILLING-CANON.md` for instructions on filling canon.json before use. Every value is verified by the human maintainer against the GitHub API or official release pages — never generated by an agent.
